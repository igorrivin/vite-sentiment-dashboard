# The Independent Set Inverse Problem for Bounded Treewidth Graphs

## Problem Statement

**Which positive integers arise as the number of independent sets of a graph with treewidth at most $k$?**

This is a bounded-treewidth generalization of the classical IS inverse problem. For trees (treewidth 1), it is conjectured that all but finitely many positive integers are achievable, but no density result is known.

## Difficulty Hierarchy

| Treewidth | Achievable (of 10,000) | Mechanism |
|-----------|----------------------|-----------|
| $= 0$ (edgeless) | 13 (0.1%) | Powers of 2 only |
| $\leq 1$ (forests) | 9,269 (92.7%) | Tree DP recurrences + multiplicative closure |
| $\leq 2$ (series-parallel) | **10,000 (100%)** | SP algebra (see below) |
| $\leq 3$ | 10,000 (100%) | Follows from tw $\leq 2$ |
| $\leq 4$ | All integers (proven) | Chain gadget / continued fraction |
| unbounded | All integers (trivial) | Threshold graphs / binary expansion |

## Main Result (Computational)

**Theorem (verified for $n \leq 10{,}000$).** Every integer $n \geq 3$ is the number of independent sets of some series-parallel graph (treewidth $\leq 2$). Combined with the empty graph ($i = 1$) and a single vertex ($i = 2$), every positive integer is achievable at treewidth $\leq 2$.

**Conjecture.** Every integer $n \geq 1$ is the number of independent sets of some graph with treewidth $\leq 2$.

This makes the treewidth $\leq 3$ question (originally posed as the hard case between the open tree conjecture and the proven tw $\leq 4$ result) **trivially resolved**: treewidth 2 already suffices.

## Part I: The Algebraic Framework

### Series-parallel graphs as matrix algebra

A two-terminal series-parallel (SP) graph $G$ with terminals $s, t$ has an **IS signature** $(f_{00}, f_{10}, f_{01}, f_{11})$, where $f_{ij}$ counts independent sets with $s$ in state $i$ and $t$ in state $j$. Arrange this as a $2 \times 2$ matrix:

$$M_G = \begin{pmatrix} f_{00} & f_{01} \\ f_{10} & f_{11} \end{pmatrix}$$

The two SP composition operations become:

- **Series composition** (identify inner terminals): $M_{G_1 ; G_2} = M_{G_1} \cdot M_{G_2}$ (matrix product)
- **Parallel composition** (identify both terminals): $M_{G_1 \| G_2} = M_{G_1} \circ M_{G_2}$ (Hadamard/entrywise product)

The **IS count** is the quadratic form:

$$i(G) = \mathbf{1}^T M_G \mathbf{1}, \qquad \mathbf{1} = \begin{pmatrix}1\\1\end{pmatrix}$$

### The Fibonacci matrix

The single edge $s$-$t$ has signature:

$$M_E = \begin{pmatrix}1&1\\1&0\end{pmatrix}$$

This is the **Fibonacci matrix**. As a Möbius transformation, it acts as $z \mapsto 1 + 1/z$, with fixed point $\varphi = (1+\sqrt{5})/2$ (the golden ratio). It satisfies $M_E^2 = M_E + I$ (the Fibonacci recurrence as a matrix equation), has determinant $-1$, and lives in $GL_2(\mathbb{Z})$.

Series powers give paths: $M_E^k$ has entries $(F_{k+1}, F_k; F_k, F_{k-1})$ and IS count $F_{k+2}$ (Fibonacci numbers).

### The generated set

Let $\mathcal{S}$ be the smallest set of $2 \times 2$ nonneg integer matrices containing $M_E$ and closed under both matrix product and Hadamard product. Then:

$$\{ i(G) : G \text{ is a series-parallel graph} \} \supseteq \{ \mathbf{1}^T M \mathbf{1} : M \in \mathcal{S} \}$$

**Computational finding:** $\{ \mathbf{1}^T M \mathbf{1} : M \in \mathcal{S} \} \supseteq \{3, 4, 5, \ldots, 10000\}$.

### The bisemigroup structure

The set $\mathcal{S}$ is closed under two associative binary operations with:

- **Different identities**: $I = \begin{pmatrix}1&0\\0&1\end{pmatrix}$ for matrix product, $J = \begin{pmatrix}1&1\\1&1\end{pmatrix}$ for Hadamard product.
- **No distributive law**: neither operation distributes over the other.
- **Neither identity in $\mathcal{S}$** (since $\mathcal{S}$ is generated by $M_E$ alone).

This is not a ring, semiring, or any standard algebraic structure — it is a set with two independent semigroup structures (a **bisemigroup**).

## Part II: The Iterated Function System

### Linearization on $\mathbb{Z}^4$

Identify each $2\times 2$ matrix with a vector $(a,b,c,d) \in \mathbb{Z}^4$ where $M = \begin{pmatrix}a&b\\c&d\end{pmatrix}$. The two operations (applied with $M_E$) become **linear maps** on $\mathbb{Z}^4$:

**Right-multiply by $M_E$ (series with edge):**
$$R: (a,b,c,d) \mapsto (a+b,\; a,\; c+d,\; c)$$

As a $4 \times 4$ matrix: $R = F \oplus F$, block-diagonal with the Fibonacci matrix acting independently on $(a,b)$ and $(c,d)$.

**Hadamard with $M_E$ (parallel with edge):**
$$P: (a,b,c,d) \mapsto (a,\; b,\; c,\; 0)$$

This is **projection** — it simply kills the $(1,1)$ entry.

The IS count is the linear functional $\varphi(a,b,c,d) = a+b+c+d$.

### Properties of the generators

| | $R$ (series) | $P$ (parallel) |
|--|--|--|
| Type | Expanding | Contracting (rank 3) |
| Eigenvalues as $4\times 4$ | $\varphi, \varphi, -1/\varphi, -1/\varphi$ | $1, 1, 1, 0$ |
| Determinant | $1$ ($\in SL_4(\mathbb{Z})$) | $0$ (not invertible) |
| Effect on IS count | $\varphi \mapsto \varphi + (a+c)$ | $\varphi \mapsto \varphi - d$ |

### The orbit

Starting from $v_0 = (1,1,1,0)$ (the edge signature), the orbit is:

$$\mathcal{O} = \{ T_{\sigma_k} \circ \cdots \circ T_{\sigma_1}(v_0) : \sigma_i \in \{R, P\},\; k \geq 0 \}$$

This is an **iterated function system** (IFS) on $\mathbb{Z}^4$ with one expanding and one contracting generator. The interplay of expansion (Fibonacci growth) and contraction (coordinate projection) creates the arithmetic diversity needed to hit all integers.

**Claim:** $\varphi(\mathcal{O}) \supseteq \mathbb{Z}_{\geq 3}$.

**Important: Sequential vs. branching IFS.** The full SP closure allows combining arbitrary previously generated matrices (not just iterating with $M_E$), giving a branching/tree-structured IFS rather than a sequential one.

### The sequential collapse (Gemini observation)

Both $R$ and $P$ are block-diagonal with the same $2 \times 2$ partition:

$$R = F \oplus F, \qquad P = I_2 \oplus E, \qquad E = \begin{pmatrix}1&0\\0&0\end{pmatrix}$$

So the top pair $(a,b)$ and bottom pair $(c,d)$ **never mix**. The top system (generated by $F$ and $I_2$) is trivial: $(a,b) = (F_{n+2}, F_{n+1})$ where $n$ is the total number of $R$'s applied.

The bottom system (generated by $F$ and $E$) collapses via the identity $E F^k E = F_{k+1} \cdot E$. This means any word with multiple $E$'s reduces to a **Fibonacci scalar** times a simpler word: every sequential orbit element has IS count of the form

$$\varphi = F_{n+3} + C \cdot F_{a+2}$$

where $C$ is a product of Fibonacci numbers.

**Consequence:** The **sequential IFS achieves only ~34% of integers** up to 1000 (337/998). Even with multiplicative closure (disjoint union), only 77% (773/1000). The sequential orbit is far too sparse for universality.

### Where the universality comes from

The missing integers are filled by **pairwise Hadamard products** of orbit elements — combining two *different* generated matrices entrywise. This is a **bilinear** operation (not linear), and it escapes the $EF^kE$ collapse entirely.

The universality engine is the iteration:
1. **Sequential products** (matrix multiplication) → structured but sparse signature matrices
2. **Pairwise Hadamard products** → entrywise multiplication of two different signatures, creating new entry patterns
3. **Repeat** → sequential products applied to Hadamard products, and vice versa

After just 4 iterations of this process, all integers 3..10,000 are achieved.

## Part III: Toward a Proof

### Sequential vs. branching: what's needed for what

A crucial distinction: the elements of the bisemigroup $\mathcal{S}$ are not **words** (sequences of operations) but **expression trees** (binary trees with leaves $M_E$ and internal nodes labeled $\cdot$ or $\circ$). With a single binary operation, every element is a word and concatenation captures all products. With **two** independent operations, the free object is the set of expression trees, which is fundamentally richer.

The **sequential orbit** (applying $R$ or $P$ one at a time to a running state — i.e., always combining the accumulated result with a fresh $M_E$) corresponds to **left-leaning trees**: $(\cdots((M_E \star M_E) \star M_E) \star \cdots)$. This is a proper subset of all expression trees.

The **full bisemigroup** includes trees like $(M_E \cdot M_E) \circ (M_E \cdot M_E \cdot M_E)$ — a Hadamard product of two independently constructed matrices. No sequential walk produces this, because the sequential model can only Hadamard the running state with a fresh $M_E$, never with another complex matrix.

For **matrix multiplication alone**, concatenation suffices: $M_1 \cdot M_2$ from sequential walks $w_1, w_2$ equals the concatenated walk $w_1 w_2$. The branching issue is specific to the **Hadamard product** — it is a second binary operation that requires combining two independently built sub-expressions.

However, this distinction has concrete consequences for achievable density:

| Goal | Required structure | Status |
|------|-------------------|--------|
| **Positive density** | Full bisemigroup (branching Hadamard) | Expected; proof requires arithmetic circuit analysis |
| **All integers (surjectivity)** | Full bisemigroup (branching Hadamard) | Verified to 10,000; proof open |

**Critical finding: the sequential orbit has density zero.** Despite hitting all residue classes mod $p$ for every prime $p$ tested, the sequential orbit thins out like the primes:

| Range | Sequential density |
|-------|-------------------|
| $[3, 100]$ | 72% |
| $[3, 500]$ | 46% |
| $[3, 1000]$ | 34% |
| $[3, 5000]$ | 19% |
| $[3, 10000]$ | 13% |

This is the **prime number analogy**: the primes are equidistributed mod $m$ for all $m$ but have density $\sim 1/\log n \to 0$. Similarly, the sequential orbit is equidistributed mod $p$ (no local obstructions) but has density tending to zero. Equidistribution mod $p$ is **necessary but insufficient** for positive density — one also needs sufficient absolute density surviving the sieve.

**Positive density genuinely requires branching.** The pairwise Hadamard products (combining two independently constructed matrices) are essential even for positive density, not just for surjectivity. The branching Hadamard operation amplifies the sparse sequential orbit to 100% density in 4 iterations.

### Mod-$p$ structure: the full closure theorem

**Theorem (verified for $p = 2, 3, 5, 7, 11$; proof sketch below).** The full bisemigroup closure of $M_E$ in $M_2(\mathbb{F}_p)$ under matrix product and Hadamard product is:

$$S_p = M_2(\mathbb{F}_p)$$

That is, **every** $2 \times 2$ matrix over $\mathbb{F}_p$ is in the closure. This is far stronger than $\varphi$-equidistribution — it says the bisemigroup has no local obstructions whatsoever.

**Proof sketch (GPT, March 2026).** Define $K_u := \begin{pmatrix}u&1\\1&0\end{pmatrix}$. Then $K_1 = E$ and $K_2 = E^2 \circ E$ (Hadamard). Since $\det(E) = -1 \neq 0$, $E \in GL_2(\mathbb{F}_p)$, and $E^{-1} \in S_p$ (as a power of $E$). The recurrence $K_{u+1} = K_2 \cdot E^{-1} \cdot K_u$ gives all $K_u$. From these, construct upper/lower unitriangular matrices $U_t = K_{t+1} E^{-1}$, $L_t = E^{-1} K_{t+1}$, diagonal matrices via Hadamard with $I$, and the permutation matrix $P = E \circ E^{-1}$. Any matrix with nonzero upper-left entry factors as $L_{c/a} D_a D'_{d-bc/a} U_{b/a}$; remaining cases use $P$ and zero factors.

**Important correction:** Our earlier claim that "the orbit is a **proper subset** of $(\mathbb{Z}/m\mathbb{Z})^4$" was about the **sequential** orbit (IFS applying $R$ or $P$ one at a time). The sequential orbit IS a proper subset (32.7% of $M_2(\mathbb{F}_7)$, decreasing with $p$). The **full bisemigroup** (pairwise products of arbitrary elements) fills everything.

**Caveat for sieve methods.** The proof requires $E^{-1} = E^{\text{ord}(E)-1}$, where $\text{ord}(E)$ divides $|GL_2(\mathbb{F}_p)| = p(p-1)^2(p+1)$. So the expression trees realizing arbitrary matrices mod $p$ have depth **polynomial in $p$**. This means the local theorem is existential per prime, not uniform: the chain lengths grow with $p$, ruling out the uniform-in-$p$ error bounds that sieve methods require. The archimedean problem (cofinite linear form on $\mathcal{R}$) is genuinely independent of the local theory — one cannot bootstrap from mod $p$ surjectivity to surjectivity over $\mathbb{Z}$.

### Exact formula for the sequential functional (Gemini)

The block-diagonal structure $R = F \oplus F$, $P = I_2 \oplus E$ (where $E = \begin{pmatrix}1&0\\0&0\end{pmatrix}$) causes the top pair $(a,b)$ and bottom pair $(c,d)$ to **decouple completely**. The top block ignores $P$ entirely (since $P_{\text{top}} = I_2$); the bottom block collapses via the identity $EF^kE = F_{k+1} \cdot E$.

For a sequential word $X = R^{a_1} P R^{a_2} P \cdots P R^{a_{k+1}}$ (with $k$ projections and $N = \sum a_i$ total $R$'s), the IS count evaluates exactly to:

$$f(X) = F_{N+3} + F_{a_1+2} \cdot F_{a_{k+1}+1} \cdot \prod_{i=2}^{k} F_{a_i+1}$$

(Verified computationally for 9,330 words with 0 mismatches.)

This has the structure $f = F_{N+3} + C$ where $C$ is a product of Fibonacci numbers. The first term is a pure Fibonacci number (from the top block); the second is a Fibonacci product (from the collapsed bottom block).

### Sequential equidistribution (Gemini's transfer operator blueprint)

For the **sequential** random walk, Gemini's transfer operator approach proves **equidistribution mod $p$** (but NOT positive density):

**Step 1: Finite state space.** Modulo a prime $p$, the Fibonacci sequence is periodic with Pisano period $\pi(p)$. The value $f(X) \bmod p$ depends only on the exponents modulo $\pi(p)$. The random walk on words becomes a finite Markov chain.

**Step 2: Affine decomposition.** The functional decomposes as an **additive shift** $B = F_{N+3} \bmod p$ (determined by total $R$-count) plus a **multiplicative accumulator** $A = F_{a_1+2} \cdot F_{a_{k+1}+1} \cdot \prod F_{a_i+1} \bmod p$ (determined by the gap structure). This has the form $f \equiv A + B \pmod{p}$.

**Step 3: Transfer operator and spectral gap.** Model the word generation as a Markov chain on a state space tracking running values mod $\pi(p)$. The exponential sum $\mathbb{E}[\chi(f(X))]$ for a nontrivial additive character $\chi$ rewrites as the norm of a "twisted" transfer operator. Equidistribution follows from showing this twisted operator has spectral radius strictly less than the untwisted one (a spectral gap).

**Step 4: Ruling out obstructions.** The spectral gap fails only if the joint distribution of $(A, B)$ is trapped in a proper subgroup of the affine group over $\mathbb{F}_p$. Because the Fibonacci sequence generates all residues mod $p$ for $p \neq 5$ (via the Pisano period), and the multiplicative accumulator takes values across $\mathbb{F}_p^*$ for generic exponent sequences, this trapping cannot occur.

**Conclusion:** This proves $f(X) \bmod p$ is equidistributed for all primes $p$ (including $p = 5$, verified computationally). But equidistribution mod $p$ is a **necessary condition** for positive density, not a sufficient one. The sequential orbit has $\sim N^{1.44}$ words producing values up to $N$ (since $2^L$ words of length $L$ produce values up to $\sim \varphi^L$), but the values themselves collide heavily — the number of **distinct** values grows much slower, yielding density zero.

### Arithmetic circuits and the branching bisemigroup (Gemini)

For the **full bisemigroup** (branching Hadamard products), the sequential transfer operator framework breaks down entirely. The correct framework is **arithmetic circuits** (straight-line programs) over the Fibonacci numbers.

#### Why the linear collapse fails for trees

The collapse identity $EF^kE = F_{k+1} E$ is a strictly **linear** phenomenon — it relies on the state always being a single matrix, with $P$ acting as a linear operator on that state. The moment Hadamard product acts as a full binary operation between two arbitrary states, we move from linear combinations to polynomials:

- If $M_1$ has entries linear in Fibonacci numbers (degree 1), and $M_2$ likewise, their Hadamard product $M_1 \circ M_2$ has entries of degree 2 (e.g., $F_5 \cdot F_8$).
- Hadamard that result with another degree-2 matrix → degree 4.
- In an expression tree of depth $D$, entries are polynomials of degree up to $2^D$.

The linear rank-collapse of $E$ is powerless to stop this exponential explosion in algebraic degree.

#### The sequential/tree dichotomy

| | Sequential semigroup (words) | Bisemigroup (trees) |
|--|--|--|
| **Algebraic model** | Deterministic finite automaton / random matrix product | Arithmetic circuit / straight-line program |
| **Operations** | $\cdot$ nodes are linear mixers | $\cdot$ nodes are linear mixers, $\circ$ nodes are nonlinear polynomial multipliers |
| **Degree of entries** | $O(1)$ (Fibonacci products) | $2^D$ (exponential in tree depth) |
| **Transfer operator** | Linear, finite state mod $p$ | Nonlinear; no finite-state reduction |

#### Proving equidistribution in the tree model

Three ingredients are needed:

1. **Algebraic independence.** Show that expression trees don't inadvertently satisfy hidden polynomial identities mod $p$. Because $\cdot$ mixes coordinates linearly and $\circ$ multiplies them, the polynomials generated at tree roots rapidly become algebraically independent. This is equivalent to **Polynomial Identity Testing (PIT)** for arithmetic circuits — notoriously difficult, tightly coupled with circuit complexity, and entangled with classes like $\text{Mod}_p\text{L}$.

2. **Schwartz-Zippel.** A nonzero polynomial of degree $d$ in $n$ variables over $\mathbb{F}_p$ has at most $d/p$ fraction of roots. For trees of depth $D$ with $p \gg 2^D$, the polynomial hits essentially all residues mod $p$.

3. **Branching exponential sums.** Apply Weyl's criterion to the tree. Instead of the spectral gap of a linear transfer operator, bound the decay of exponential sums over trees. The variance of the distribution mod $p$ decays exponentially with tree depth, driven by the Hadamard product continuously squaring the state space.

#### The three barriers to rigorous proof (Gemini's assessment)

Gemini assessed the tractability of this program as follows:

**Barrier 1: Degree-characteristic.** Standard exponential sum bounds (Weil bounds, Deligne's theorem, the Bourgain-Gamburd machine) require the polynomial degree $d < p$. But an expression tree of depth $D \approx \log_2 p$ already produces degree $2^D > p$. Once $d \geq p$, the polynomial "wraps around" $\mathbb{F}_p$ chaotically and classical algebraic geometry estimates give no cancellation.

**Barrier 2: PIT and circuit complexity.** Proving that the expression trees don't get trapped in a lower-dimensional algebraic subvariety mod $p$ is equivalent to PIT for arithmetic circuits with Hadamard product. This is deeply entangled with open problems in circuit complexity. Rigorously proving algebraic independence for these specific trees would likely require novel circuit lower bounds.

**Barrier 3: No branching expander machine.** The Bourgain-Gamburd machine works because sequential matrix multiplication acts as a linear operator, enabling Lyapunov exponents and linear transfer operators. In the bisemigroup, the measure at depth $n+1$ evolves via a **nonlinear** convolution:

$$\mu_{n+1} = \alpha\, (\mu_n * \mu_n) + \beta\, (\mu_n \circ \mu_n)$$

This is a nonlinear dynamical system on the simplex of probability measures. There is currently no generalized Bourgain-Gamburd expansion machine for nonlinear measure evolutions of this type.

#### A tractable pivot: entropy and coupling (Gemini's suggestion)

Since behavior mod $p$ lives on a **finite** state space ($|M_2(\mathbb{F}_p)| = p^4$), one can study the evolution of Shannon entropy rather than tracking algebraic degree:

1. **Entropy growth.** Show that taking the Hadamard product of two independent measures $\mu \circ \mu$ strictly increases entropy, unless the measure is already uniform (or trapped in a degenerate ideal).

2. **Coupling.** Because tree operations continuously mix independent samples, build a probabilistic coupling argument showing the total variation distance to uniform contracts by $(1 - \epsilon)$ at each tree level.

By abandoning the infinite-dimensional algebraic structure and focusing on finite nonlinear mixing of probability weights mod $p$, the equidistribution problem transitions from intractable arithmetic geometry to potentially solvable **ergodic theory of nonlinear Markov operators**.

#### The sieve gap: equidistribution ≠ surjectivity

Even if the entropy/coupling approach succeeds in proving equidistribution mod $p$ for all primes $p$, this is **still insufficient** for surjectivity (and potentially even for positive density). The reason is quantitative: sieve methods (large sieve, Selberg sieve, affine sieve) require not just that the distribution mod $p$ is close to uniform, but that the **error terms decay fast enough in $p$** for the product over primes to converge.

Specifically, if the total variation distance to uniform mod $p$ is $\delta(p)$, the sieve needs $\sum_p \delta(p) / p < \infty$ (or similar convergence conditions) to extract a positive density lower bound. Ergodic methods are notoriously bad at providing such quantitative control: they typically give qualitative convergence ("the measure converges to uniform") or exponential mixing rates $\delta \sim e^{-cn}$ in the number of iterations $n$, but with constants $c = c(p)$ that depend on $p$ in opaque ways. Controlling $c(p)$ uniformly as $p \to \infty$ is a hard analytic problem.

This is precisely the situation in Apollonian packings:
- **Positive density** (Bourgain-Fuchs 2011): achieved because the sieve error terms are good enough for a lower bound, leveraging the specific algebraic structure of the Apollonian group (thin subgroup of $O(3,1;\mathbb{Z})$) and deep results on expanders in $SL_2(\mathbb{F}_p)$.
- **Surjectivity** (local-to-global conjecture): **still open** after 15+ years, because the error terms are not sharp enough for an exact count. The gap between "positive density" and "all integers" remains resistant to all known techniques.

For the bisemigroup problem, the situation is arguably harder: we lack even the group structure that makes the Apollonian sieve work (since $P$ is non-invertible), and the nonlinear measure evolution $\mu_{n+1} = \alpha(\mu_n * \mu_n) + \beta(\mu_n \circ \mu_n)$ has no precedent in the sieve literature.

**Bottom line:** Equidistribution mod $p$ (even with good rates) → positive density (via sieve) → surjectivity is a chain with two hard gaps. The first gap requires uniform-in-$p$ error bounds from the ergodic theory; the second gap is the local-to-global problem, which is open even in much better-understood settings.

### Analogy with Apollonian packings

The structure parallels the Apollonian gasket, where similar methods yield positive density:

|--|--|--|
| **Space** | Descartes quadruples $\in \mathbb{Z}^4$ | Matrix entries $\in \mathbb{Z}^4$ |

### The density and surjectivity gaps

Both positive density and surjectivity require the **branching Hadamard products**. In the full bisemigroup, one can Hadamard-multiply two arbitrary orbit elements: $(a_1,b_1,c_1,d_1) \circ (a_2,b_2,c_2,d_2) = (a_1 a_2, b_1 b_2, c_1 c_2, d_1 d_2)$. The IS count of the result involves sums of products of Fibonacci numbers — a much richer arithmetic class than the sequential orbit, which is trapped in $F_{N+3} + \text{(Fibonacci product)}$.

The sequential orbit has density zero (like the primes) despite perfect equidistribution mod every prime. This means:
- **Equidistribution mod $p$**: provable for the sequential walk, but insufficient for positive density
- **Positive density**: requires branching Hadamard to amplify the word count beyond the $N^{1.44}$ barrier
- **Surjectivity**: requires branching Hadamard to fill all gaps (verified to 10,000 in 4 iterations)

Computationally, 4 iterations of the series/Hadamard cycle (combining arbitrary pairs) achieve all integers 3..10,000. Proving surjectivity or even positive density rigorously — showing the branching Hadamard products produce enough new values — is the local-to-global problem, analogous to the still-open Apollonian conjecture.

## Part IIIb: Alternative Proof Strategies (GPT, March 2026)

The sieve/PIT/expander route may be unnecessarily heavy. Several more direct, combinatorial approaches may bypass the hard analytic barriers entirely.

### Lemma A: Transpose closure ($\mathcal{R} = \mathcal{C}$)

Since $M_E$ is symmetric and $(A \circ B)^T = A^T \circ B^T$, $(AB)^T = B^T A^T$, any expression tree can be "mirrored" (swap children at each $\cdot$-node) to produce the transpose. So $\mathcal{S}$ is closed under $M \mapsto M^T$.

Define $\operatorname{row}(M) = \mathbf{1}^T M$ and $\operatorname{col}(M) = M^T \mathbf{1}$. Then $\operatorname{col}(M) = \operatorname{row}(M^T)$, and since $M^T \in \mathcal{S}$:

$$\mathcal{R} := \{\operatorname{row}(M) : M \in \mathcal{S}\} = \{\operatorname{col}(M) : M \in \mathcal{S}\} =: \mathcal{C}$$

This is a **theorem**, not a finite-depth artifact. (Verified computationally at depth 4.)

### Lemma B: Dot-product identity (2D reduction)

For $M = \begin{pmatrix}a&b\\c&d\end{pmatrix}$, using $\operatorname{col}(M) := (a+c,\; b+d)$ and $\operatorname{row}(N) := (a'+b',\; c'+d')$:

$$\boxed{\varphi(MN) = \operatorname{col}(M) \cdot \operatorname{row}(N) = (M^T \mathbf{1}) \cdot (\mathbf{1}^T N)}$$

**Proof:** $\varphi(MN) = \mathbf{1}^T MN \mathbf{1} = (M^T \mathbf{1})^T (N \mathbf{1})$. Since $\mathcal{R} = \mathcal{C}$:

$$\varphi(\mathcal{S}) \supseteq \{u \cdot v : u, v \in \mathcal{R}\}$$

### The $(2, 1)$ shortcut

Since $\operatorname{col}(M_E) = (2, 1)$, for any $N \in \mathcal{S}$ with $\operatorname{row}(N) = (x, y)$:

$$\varphi(M_E \cdot N) = 2x + y$$

So surjectivity reduces to:

> **Show that $L := \{2x + y : (x, y) \in \mathcal{R}\}$ is cofinite** (i.e., misses only finitely many positive integers).

This cleanly separates the "graph theory / SP algebra" part (what $\mathcal{R}$ looks like) from the "number theory" part (how a fixed linear form hits $\mathbb{N}$).

### The Fibonacci transform on row vectors

Right-multiplication by $M_E$ acts on row vectors as the **Fibonacci map**:

$$\operatorname{row}(M \cdot M_E) = \operatorname{row}(M) \cdot M_E = (x + y,\; x)$$

This map preserves $\gcd(x, y)$, and under it $2x + y \mapsto 2(x+y) + x = 3x + 2y$. So if $(x, y) \in \mathcal{R}$ achieves $2x + y = t$, then the Fibonacci iterate achieves $3x + 2y = t + x$. Repeated application gives $2x_n + y_n = F_{n+2} x + F_{n+1} y$ (Fibonacci-weighted combinations), sweeping values that grow like $\varphi^n$.

**Key implication for the proof:** if the Hadamard product produces enough **seed vectors** $(x, y)$ with modest $y$, the Fibonacci transform propagates them into families where $2x + y$ sweeps intervals. The problem reduces to finding a dense enough set of seeds.

### Strategy 2: Interval growth bootstrap

Rather than proving global equidistribution, prove that the achievable set contains arbitrarily long **initial intervals** $[3, N]$ with $N \to \infty$:

- **Round 0**: series/Hadamard closure gives some interval $[3, N_0]$.
- **Inductive step**: if you have $[3, N]$, then after one round of closure you have $[3, N']$ with $N'$ polynomially larger (perhaps $N' \gtrsim N^2$ from the dot-product identity).
- **Iterate**.

Given the empirical observation that 4 iterations fill $[3, 10000]$, this is the natural theoretical version. The dot-product identity from Strategy 1 is exactly the tool that yields such interval-growth lemmas. This bypasses any local-to-global / sieve issues entirely.

### Strategy 3: Classify small-entry regimes

Certain boundary counts being $0$ or $1$ force rigid SP graph structure. For example, matrices with $b = c = 1$ and $d = 0$ force $a$ to be a power of $2$ (from the "book of triangles" structure). Systematically classifying TTSP graphs where one or more entries is $0$ or $1$ yields:

- **Provable invariants** for large subfamilies, shrinking the search space.
- **Explicit parametrizations** that make dot-product representations easier.

This is provable by SP decomposition induction (split at the last series/parallel operation) and avoids heavy external machinery.

### Strategy 4: Check if the tw $\leq 4$ continued-fraction gadget embeds at tw $\leq 2$

The proven tw $\leq 4$ universality uses a continued-fraction recurrence corresponding to multiplying matrices $\begin{pmatrix}m&1\\1&0\end{pmatrix}$ (continuants). If this can be realized as a **two-terminal SP composition** (keeping treewidth $\leq 2$), it would immediately prove surjectivity.

This amounts to implementing the continuant matrices as SP signatures — which overlaps with the main conjecture. Even a partial result ("you can implement all $m$ in some dense subset") plus an interval-bootstrap could finish surjectivity.

### Strategy 5: Holant / tensor-network literature

The algebraic framework is exactly "binary signature closure under contraction (series) and pointwise product (parallel)" — the **Holant / tensor network** closure operation, specialized to the hard-core edge constraint $M_E = \begin{pmatrix}1&1\\1&0\end{pmatrix}$. The question "which binary signatures are realizable from a given generator under SP composition?" is standard in that literature. An existing characterization of realizable signatures could collapse the circuit-complexity concern entirely.

### Computational validation of the dot-product strategy

The dot-product identity was tested computationally with striking results:

**Row/col sum sets:** After 4 iterations of bisemigroup closure (2,594 signatures), $\mathcal{R} = \mathcal{C}$ (the sets are identical, 1,902 vectors). However, they do **not** form thick rectangles: coverage of $[1..10]^2$ is only 25%, and there is no $K > 1$ with $[2..K] \times [1..K] \subseteq \mathcal{R}$.

**Despite this, dot products are nearly surjective:**

| Range | Dot product gaps (from 4-iteration closure) |
|-------|---------------------|
| $[1, 10000]$ | Only $\{1, 2, 3, 4, 6, 9\}$ |
| $[10001, 40000]$ | **None** |
| $[40001, 50000]$ | 72 gaps (finite-depth artifact) |

The gaps in $[1, 10000]$ are exactly $\{1, 2, 3, 4, 6, 9\}$. Since $3, 4, 6, 9$ are all directly achievable as IS counts of small SP graphs, **every integer in $[3, 10000]$ is covered** by the dot-product route alone. The gaps near 43,000–50,000 are artifacts of the finite (2,594-signature) closure; they disappear with deeper iteration (the full bisemigroup achieves all integers $3..10{,}000$ and presumably beyond).

**Per-column-vector thresholds:** For fixed col vector $(a, b)$, the achievable dot products $\{ax + by : (x, y) \in \mathcal{R}\}$ become **consecutive** above a threshold:

| Col vector | Consecutive from |
|-----------|-----------------|
| $(4, 3)$ | 137 |
| $(3, 1)$ | 149 |
| $(3, 2)$ | 165 |
| $(2, 1)$ | 174 |

The union of these sets covers all integers $\geq 5$ (except 6 and 9, achieved directly).

**Row-sum structure:** Paths $P_n$ (series of $n$ edges) give row sums $(F_{n+2}, F_{n+1})$ — consecutive Fibonacci pairs. Hadamard products of paths give row sums involving Fibonacci products. The row-sum set has Fibonacci-scale gaps (max gap $\sim 64$ for $y = 1$, but decreasing gap ratios for larger $y$), explaining why the set is too sparse for thick rectangles but rich enough for dot-product coverage.

**Revised attack plan:** The "thick rectangle" hypothesis is FALSE, but surjectivity holds anyway. The proof should target the **line-hitting property** of $\mathcal{R}$:

> **Target Lemma (Cofinite Linear Form).** There exists an explicit $T$ such that for every $n \geq T$, there is a reachable row vector $(x, y) \in \mathcal{R}$ with $2x + y = n$.

Then small $n < T$ are handled by a finite list of explicit SP graphs (computationally verified).

**Proof strategy for the Target Lemma:**

1. **Seeds from Hadamard products.** Two explicit seed families:
   - **Paths:** $r(E^n) = (F_{n+2}, F_{n+1})$, giving $2x + y = 2F_{n+2} + F_{n+1}$.
   - **Parallel paths:** $r(E^m \circ E^n) = (F_{m+n+1}, F_{m+n-1})$ (verified; uses Fibonacci identity $F_{m+1}F_{n+1} + F_mF_n = F_{m+n+1}$), giving $2x + y = 2F_{s+1} + F_{s-1}$ for $s = m + n$.
   Both families produce Fibonacci-spaced values (growth $\sim \varphi^n$), too sparse alone.
2. **Fibonacci propagation.** The map $(x, y) \mapsto (x+y, x)$ (right-multiplication by $M_E$) sends $2x+y = t$ to $2(x+y)+x = t + x$. Repeated application creates Fibonacci-spaced sequences of achievable values.
3. **Gap filling.** Show that the seeds are dense enough (in the $2x + y$ sense) that the Fibonacci-propagated families **interleave** to cover all integers above $T$.
4. **Mine witnesses, not just values.** For each $n \leq N$, store a minimal expression tree realizing $n$. Cluster realizations by top-level template. If most integers in a long interval come from only a few templates, that structure is the proof trying to reveal itself.

This is a 1D interval-coverage argument, not a 2D rectangle condition. The "fibers" $\mathcal{R}_y := \{x : (x, y) \in \mathcal{R}\}$ and how they grow under the two operations are the key objects to analyze.

**Remark on the overall proof architecture.** The local theory is now **completely solved**: $S_p = M_2(\mathbb{F}_p)$ for all primes $p$, so there are no mod-$p$ obstructions whatsoever. The remaining problem is purely **archimedean** — showing the integer row-sum vectors are dense enough in the $2x + y$ direction. The PIT/expander/sieve machinery (Part III above) is context for why the problem is hard in general, but the actual proof target is the cofinite linear form lemma, which is a statement in elementary additive number theory about a structured subset of $\mathbb{N}^2$.

## Part IV: Prior Results and Context

### The tw $\leq 4$ construction (proven universal)

**(Due to OpenAI o3, March 2026.)** The "chain gadget" $C(m_1, \ldots, m_k)$ is a caterpillar-like graph with IS count following the continued-fraction recurrence:
- $A_i = m_i \cdot A_{i-1} + B_{i-1}$, $B_i = A_{i-1}$
- $(A_0, B_0) = (1, 1)$
- IS count $= A_k + B_k$

The Euclidean algorithm guarantees that for ANY target $T$, parameters $m_1, \ldots, m_k$ exist (with $k = O(\log T)$) such that $A_k + B_k = T$. Each gadget has treewidth $\leq 2$; adding the spine gives treewidth $\leq 4$.

**Consequence:** Every positive integer is the IS count of a graph with treewidth $\leq 4$. (Proven.)

### The tw $\leq 3$ verification

**(Also due to OpenAI o3.)** GPT constructed explicit graphs with treewidth $\leq 3$ hitting specific large targets using the "Combine gadget" (disjoint union + apex vertices). Example: a 243-vertex graph with IS count $= 46084710702216534918083727566658949233732143042229436416$, verified independently.

The approach uses: factor the target, realize each factor via a tw $\leq 2$ component or a single Combine step from tw $\leq 2$ to tw $\leq 3$, take disjoint unions.

### The tree IS conjecture

For trees (treewidth 1), it is conjectured that all but finitely many positive integers occur as $i(T)$ for some tree $T$. This is **open** — no density result is known. Our data shows trees miss ~7.3% of integers up to 10,000 (including primes 7, 11, 19, 29, 31, ...).

The jump from tw $\leq 1$ to tw $\leq 2$ closes all gaps. The mechanism: parallel composition (Hadamard product) has no tree analogue and is what provides the arithmetic diversity.

### Linek's theorem

Linek (1989): every positive integer is the IS count of some **bipartite graph**. But with no treewidth bound. Our conjecture strengthens this to treewidth $\leq 2$ (though not all SP graphs are bipartite).

## Significance

1. **Potential first density result for bounded-treewidth IS inverse problem.** No density result is known even for trees. The full mod-$p$ closure theorem ($S_p = M_2(\mathbb{F}_p)$) eliminates all local obstructions, making the remaining problem purely archimedean.

2. **Complete local picture.** The bisemigroup closure mod $p$ is all of $M_2(\mathbb{F}_p)$ — far stronger than $\varphi$-equidistribution. This means the PIT/expander/sieve machinery is not needed for the local theory; it is already solved by an elementary algebraic argument (LDU factorization from $K_u$ matrices).

3. **Clean reduction to 1D number theory.** Via $\mathcal{R} = \mathcal{C}$ (transpose closure) and the dot-product identity $\varphi(MN) = \operatorname{col}(M) \cdot \operatorname{row}(N)$, the surjectivity problem reduces to: is $\{2x + y : (x, y) \in \mathcal{R}\}$ cofinite?

4. **Finite boundary algebra viewpoint.** The $2 \times 2$ terminal signatures are the $k = 2$ instance of the graph-homomorphism / Holant framework: $i(G) = \text{hom}(G, H_{\text{ind}})$ for the one-looped edge. The series/parallel operations are contraction and pointwise product of binary signatures. This is a well-studied framework (Holant clone theory, Fibonacci-gate papers) that may provide structural lemmas; see [arXiv:1811.00817](https://arxiv.org/pdf/1811.00817).
